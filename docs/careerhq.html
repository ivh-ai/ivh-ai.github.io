Project documentation

Career HQ

A personal job-search operating system: one place where a resume lives, jobs get discovered, applications get tracked, and interviews get prepped — powered by a local dashboard, a markdown data layer, and a roster of custom-built Claude skills.

  • Node.js
  • Claude Skills
  • MCP
  • Markdown

The idea

A serious job search generates chaos: versions of resumes, dozens of postings, application statuses, interview notes, company research. Career HQ turns that chaos into a system with three rules:

  • One source of truth — every piece of career data lives in plain, portable files, not scattered across apps
  • Log everything — untracked actions are lost actions; every application gets recorded automatically
  • Never fabricate — the AI can sharpen framing, but resume content, metrics, and credentials must be real, with gaps flagged honestly
In plain English

How it works

The filing cabinet

Everything — master resume, positioning, applications, interview notes — lives as readable text files on the machine. No vendor lock-in, no subscription, nothing to export later. If every other tool disappeared tomorrow, the data would still be perfectly usable.

The dashboard

A local web app reads those files and presents the whole job search at a glance: the application pipeline, statuses, and next actions. Edits made in the dashboard are written straight back to the same files.

The skills

Seven custom-built Claude skills act like specialist assistants that all share the same filing cabinet: one tailors the resume to a specific posting, one finds and scores new job listings, one tracks applications, one builds interview prep from the company's actual context, and others compare offers and research industries. Live job-board data flows in through MCP integrations with sites like Indeed and ZipRecruiter.

For technical readers

Under the hood

Markdown as database

The data layer is a structured directory of markdown files with stable conventions — a tracker table for applications, per-application field labels, and a profile folder holding the master resume and positioning docs. Structure is enforced by convention and parsers rather than a schema, keeping every record human-readable and version-controllable.

The dashboard server

A dependency-light Node.js server parses the markdown into a live web UI and writes changes back to disk. Because the parsers and the file conventions are coupled, the conventions are documented as contract: change a column, update the parser.

Skills + MCP architecture

Each skill encodes a workflow's rules once (e.g., resume-tailor reads the master resume, never invents content, and outputs a one-page tailored version) so every future session follows the same playbook. Job-board MCP connectors are discovered dynamically and merged, with web search as fallback — the system degrades gracefully to local-files-only when nothing is connected.

Key design decisions

Local-first and private

Career data is among the most personal data there is. Everything runs and stays on the local machine — which is why the public demo runs entirely on fictional sample data.

Plain text over apps

Markdown files outlive every SaaS tool and are natively readable by both humans and AI — the ideal shared memory between a person and their assistant.

Skills over prompts

Instead of re-explaining the job-search rules in every conversation, each workflow is encoded once as a skill. Consistency is the feature.

Honesty as a hard rule

The system's core constraint is written into every skill: never fabricate resume content or metrics. An AI job-search tool is only useful if recruiters can trust its output.
