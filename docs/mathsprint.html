MathSprint — Quick Math Challenge

90 seconds. As many questions as you can. A fast-paced mental-math trainer with four game modes, streak tracking, and a shared leaderboard — all delivered as one dependency-free HTML file that loads instantly.

  • JavaScript
  • HTML/CSS
  • Supabase
  • Zero dependencies

The idea

Mental math gets rusty without practice, and most practice tools feel like homework. MathSprint turns it into an arcade run: pick an operation, race a 90-second clock, and chase a high score. The design constraints:

  • Instant to play — no login, no load time, no instructions longer than a glance
  • Four focused modes — addition, subtraction, multiplication, and division, so you can train your weak spot
  • Momentum mechanics — streak tracking rewards consistency, not just speed
  • Friendly competition — a leaderboard per game mode, shared across all players
In plain English

How it works

The game loop

Each question shows a problem like 12 + 4 with three answer choices. Tap the right one and your score ticks up; wrong answers still advance to the next question — you get one attempt each, so guessing carelessly costs you. Keyboard players can press 1, 2, or 3 to answer even faster.

The clock and the streak

Every run is exactly 90 seconds. Alongside your score, the game tracks your current and best streak of consecutive correct answers — the real skill measure, since one careless tap resets it.

The leaderboard

When time's up, your score is submitted under your name to a shared cloud leaderboard — one board per game mode, so addition specialists and division wizards each get their own top ranks.

For technical readers

Under the hood

Single-file architecture

The entire game — markup, styles, and logic — ships as one HTML file with no build step, no framework, and no external dependencies. Screens (home, instructions, game, results, leaderboard, credits) are implemented as .page sections toggled by a tiny router, which keeps navigation instant and the whole game trivially portable: copy one file, and it runs anywhere a browser does.

Question generation

Problems are generated on the fly per mode, with distractor answers constructed to be plausible (near-misses rather than random numbers), which is what makes the one-attempt rule meaningful.

Leaderboard with graceful fallback

Scores post to a Supabase table through its REST API, queried per mode and ordered by score. If the Supabase configuration is absent, the game silently falls back to browser localStorage — the design degrades gracefully instead of breaking, so the file works even fully offline.

Key design decisions

One file, forever portable

No framework or bundler for a game this size — a single HTML file is faster to load, easier to host, and impossible to break with a dependency update.

Three choices, one attempt

Multiple choice keeps the pace high (no typing numbers), while the single-attempt rule keeps it honest — you can't brute-force the right answer.

90 seconds exactly

Long enough to build a rhythm, short enough to always justify "one more run" — the session length is the retention mechanic.

Per-mode leaderboards

A shared board across operations would just measure addition speed. Separate boards make each mode its own competition.
