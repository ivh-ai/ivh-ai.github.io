How to use

Using Career HQ

Career HQ is a focused job-application tracker: a kanban board for every application, an at-a-glance overview of what needs attention, and Claude behind the scenes for the resume, prep, and research work. This guide tours the interactive demo, then explains how the system runs a real search.

Quick start

Open the demo → switch between the Overview and Board tabs → drag a card between stages → open a card to log an interview round → add a job by pasting a posting URL. Everything is fictional sample data, and every change is simulated in your browser and resets on refresh.

Touring the tracker

  1. Overview — what needs attention today

    The landing tab answers "what should I do now?" Four stat cards — open applications, how many are in interview stages, offers on the table, and this week's activity — sit above a Needs attention list (every application that's overdue or missing a next action) and a stage-by-stage pipeline funnel. Click any attention item to jump straight to that card.

  2. Board — the kanban pipeline

    Every application is a card in a stage column: Saved → Applied → Screen → Interview → Final → Offer, with closed applications collapsed into an archive rail. Drag a card to another column to move its stage — in the real system that rewrites the tracker file on disk. Overdue next actions show in red, and a job with no next action is flagged.

  3. The job card

    Each card carries the details you act on without opening anything: company and role, location and salary, the source board, how many days it's sat in its current stage, a link back to the original posting, and its next action with a due date.

  4. Adding a job by URL

    Hit + Add job and paste a posting link. The real dashboard fetches the page and auto-fills the company, role, location, salary, and source; you review, fix anything, and save. Postings that block fetching fall back to quick manual entry. (In this demo the fetch is stubbed, so you fill the form in by hand.)

  5. Interview rounds

    Open any card and log each interview round — date, type (phone, technical, onsite, panel…), status, and notes. The card then shows a "Round N" badge and the date of the next scheduled round, so multi-stage interview loops stay legible at a glance.

How it runs a real job search

The board is the tracking hub; Claude is the engine for everything around it. The heavier work runs as Claude skills in chat, each writing back to the same files the board reads:

  1. Ask Claude to do the work

    "Find jobs that match my profile," "Tailor my resume to this posting," "Prep me for my final round at [company]" — each request triggers a purpose-built skill that already knows the rules of the system and its honesty constraints.

  2. Claude writes it down as plain markdown

    Skills save resumes, prep packs, and research as markdown files, and log every application and touch to the tracker — so the board and overview reflect it the moment you refresh, because both read the same files.

  3. The board keeps you honest

    Every open application is expected to carry a next action. The overview surfaces anything overdue or unattended, so nothing quietly stalls, and a weekly review walks the whole pipeline.

Why this demo is sample data

The real Career HQ runs locally on private career data. This demo swaps that for a fictional pipeline and simulates edits in your browser — drag cards, log rounds, add a job — without saving anything. For a walkthrough of the real system, get in touch.
