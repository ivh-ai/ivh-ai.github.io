COTE — Countries of the Earth

An interactive geography game: name as many of the world's 197 countries as you can. Each correct answer lights up on a 3D rotating globe in its real topographical color, and a global leaderboard is shared across every device.

  • React
  • Vite
  • D3.js
  • TopoJSON
  • Supabase

The idea

The goal was a geography quiz that felt polished, ran entirely in the browser, and required zero setup from the player. The requirements going in:

  • A 3D interactive globe that updates in real time as players name countries
  • Countries filled in with topographical colors — lush greens, desert sands, temperate grays — when guessed
  • A global leaderboard visible to all players, on any device
  • Smart spell-checking: minor typos accepted, near-misses get a "Did you mean?" suggestion
  • A clean, minimal, dark UI
In plain English

How it works

The game is three parts working together:

The globe

A 3D ball drawn on screen using math that projects flat map data onto a sphere. Drag it and the math updates, so it feels like spinning a real globe. Guess a country and its shape fills in with color, with a brief zoom to show you where it is.

The input bar

Where you type. The game checks spelling on every keystroke — no Enter key needed. An exact answer (or one within a single typo) is accepted instantly; if you're close but not quite there, a clickable "Did you mean?" suggestion appears.

The leaderboard

A small database in the cloud. When a game ends, your name and score go to that database — and everyone who opens the leaderboard sees the same live top 10, regardless of what device they're playing on.

For technical readers

Under the hood

Rendering pipeline

The globe uses D3's geoOrthographic projection to map spherical coordinates onto 2D SVG paths. A requestAnimationFrame loop runs continuously but only re-renders when the projection state is dirty. Each country is an SVG <path> whose fill is resolved from a 197-entry TOPO_COLOR lookup table keyed by country ID.

Fuzzy matching

Two Levenshtein edit-distance implementations do different jobs: a fast version with an early exit at distance 2 decides whether to accept a guess (distance ≤ 1 from any name or alias passes), while a full version computes true similarity for the "Did you mean?" feature — any unguessed country scoring ≥ 75% similarity is surfaced as a suggestion.

Leaderboard architecture

Scores live in a Supabase table accessed through its REST API directly — no SDK. Row Level Security allows public SELECT, INSERT, and DELETE but intentionally blocks PATCH, so score updates are implemented as delete-then-insert, triggered only when a returning player beats their personal best. Score formula: countries × 1000 − seconds, weighted so finding more countries always beats a faster time.

Build highlights

The game was built in phases with Claude Code — core loop first, then colors, spell checking, the cloud leaderboard, and polish. A few battles worth telling:

The gray-country bug

Guessed countries wouldn't take their color. Root cause: TopoJSON zero-pads IDs ("076" for Brazil) while the game data doesn't ("76"). One conversion — String(+d.id) — fixed all 197 countries, verified by a full simulation before launch.

Misleading suggestions

"Frnce" once suggested North Macedonia. The fast edit-distance function's early exit produced false similarity scores — fixed by adding the separate full-distance function for suggestions.

From local to global scores

The first leaderboard used the browser's localStorage, so every device only saw its own scores. It was replaced entirely with Supabase so all players share one board.

Tuning the feel

The zoom-on-guess animation went from 7× to 5× to a final 3× — the first two felt jarring. All input feedback was consolidated below the input bar so the layout never shifts.
